Programming Techniques

Object Oriented Programming

Object Oriented Languages are based on creating classes that define the attributes and behaviours for objects. Having an object means that all the data in a given objects says within itself, preventing the different parts of the object from being spread around your code or data that should be protected from being accessed. This conforms to the concept of encapsulation.

Pros

  • Due to modular coupling different parts of a project can be worked on separately.
  • The classes are often reusable.
  • The code can be easier to maintain due to a centralised coding base.

Cons

  • Can be more CPU intensive than other alternatives.
  • Can lead to bloated code.

Classes

Classes are where we define the attributes and behaviours of a group of objects (known as the class).

We can instantiate a class to create an instance of it, which is an object.

When a function is created inside of a class it is known as a method of the class.

The Constructor

The constructor is the first method of the class that is automatically run when a new instance of the class is created. These usually take some form of parameters that affect the running of the object.

Inheritance

Some classes are best written as Subclasses of a bigger class. A subclass inherits all of the attributes and methods of the Superclass it is derived from. A subclass can implement its own specific methods.

Polymorphism

A form of inheritance in which the methods of the superclass are overwritten. The Superclass is therefore sometimes abstract where the definitions for methods are there but they are empty and are expected to be overwritten in the subclass.
