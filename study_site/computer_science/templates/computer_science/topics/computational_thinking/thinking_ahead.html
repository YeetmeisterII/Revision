Thinking Ahead

Input / Process / Output

All computer systems consist of these three things at a basic level

  • Input: All data fed into the system for processing.
  • Process: What happens to inputted data.
  • Output: The data and information which is fed out of the system after processing.

Modular Design

Advantages

Easier to write and test - the fewer things a program does, the fewer things can go wrong.

It is clear from the initial design which parts of the program should interact with each other.

Different sections of the program can be assigned to different programming teams - many teams will have expertise in specific areas.

Each module performs a specific task - any future programs which need to perform the same task they can be reused.

Problem Decomposition

When you analyse and break down a problem, creating many programs which are each able to handle a single task is easier.
