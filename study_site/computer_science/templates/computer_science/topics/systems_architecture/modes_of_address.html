Modes of Address

Immediate Addressing

  • This operand is literally the value that is used for the instruction
  • Super fast
  • Nothing needs to be fetched from memory, it's already there to go
  • Limits the range of values which can be used
  • Requires values to be known ahead of time

Direct Addressing

  • The operand is a memory address which points to a location which holds the value to be used
  • Slower than immediate addressing
  • Allows for a greater range of values to be used

Indirect Addressing

  • The operand contains an address which points to a memory location which contains an address which points to another memory location which contains the data
  • Allows access to a greater range of memory addresses
  • Requires twice as many memory accesses as direct addressing

Indexed Addressing

Indexed addressing requires a new register the IR (Index Register)

  • The operand is an address to be accessed
  • This value is added to the contents of the index register (this is called the offset)
  • The result is then copied to the MAR and memory is accessed
  • Finally, the index register is incremented
  • In this way, the same command can be used multiple times accessing sequential locations in memory
