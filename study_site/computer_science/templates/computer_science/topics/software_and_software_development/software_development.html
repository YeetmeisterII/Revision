Software Development

The Waterfall Model

The waterfall model goes through five stages

Analysis

  • Feasibility study
  • Problem identified
  • Existing solutions studies
  • Feature list determined

The feasibility of solving the main problem is researched to determine whether the cost and time of solving the problem is worth it.

Deconstruct the problem into smaller parts that can be tackled actionable.

Existing solutions are studied to ensure that any work being done is not recreating the wheel.

Feature list is determined to prioritise what features are necessary and which features are additions.

Design

Implementation

Working from the plans the program is created according to specification.

Evaluation

The system is tested and bugs in the program is fixed at this stage in the development.

Maintenance

After implementation bugs may arise through general use, and are fixed in patches.
