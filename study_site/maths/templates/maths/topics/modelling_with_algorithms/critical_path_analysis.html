Critical Path Analysis

Usage

This looks at how we manage a series of activity which may interlink with each other to form a complete task. We want to perform the whole task as quickly as possible.

Activity Network

Activity networks are graphs that display the scope of activities within a task.

  • The edges represent the activities
  • Activities are lettered
  • The weights are the durations
  • The nodes represent events (moments in time between the completion of an activity and the start of another)
  • The graph is typically read from left to right
  • The network must have a single start event and a single finish event

Dummy Activities

  • A dummy activity has duration zero
  • It is shown using dotted lines

Identifying a Critical Path

The minimum time in which a project can be completed - assuming no delays - is the minimum project completion time. It is length of the longest path in the activity network. Any activity on the longest path is called a critical activity.

The Algorithm Process

Forward Pass

  • Go in order of the nodes and find the longest time taken to reach said node from the starting node
  • Continue until reaching the ending node

Backwards Pass

  • From the finishing node look at he before it
  • Deduct the shortest length of the shortest path between the nodes
  • Repeat the process until you reach the starting node

Evaluation

The forward pass finds the longest time an event could take whereas the backwards pass finds the latest time an event could take place, when there is a discrepancy between the two it tells us that the activity is not critical.

Float

Non-critical activities have 'float'. This is the amount of time by which the duration of the activity can be extended without having any effect on the minimum completion time.

The float may be divided into independent float (which is unique to this activity) and interfering float (which has an impact on the float of other activities)

Float for activity \( (i, j) = \text{ latest event time at } j - \text{ earliest event time at } i - \text{ duration } \)

Independent float for \( (i, j) = \text{ earliest event time at } j - \text{ latest event time at } i - \text{ duration } \)

\( \text{Interfering float } = \text{ Total float } - \text{ Independent float } \)
